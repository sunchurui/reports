# 2605.12894 — Beyond Cooperative Simulators

**完整标题**：Beyond Cooperative Simulators: Generating Realistic User Personas for Robust Evaluation of LLM Agents
**来源**：Harshita Chopra et al., arXiv 2605.12894, 2026-05
**调研主题**：客服对话仿真系统的 user_agent 设计 — persona 多样性

## 一句话总结

提出 Persona Policies (PPol)，通过 evolutionary program search 自动生成多样化、类真人的 user persona，解决 LLM simulator 过度配合的问题。

## 核心方法

**Persona Policies (PPol)** — plug-and-play 控制层，附加到现有 user simulator 的 system prompt 上，不改变底层 benchmark task。

**核心思路**：将 persona 生成建模为 **evolutionary program search**（基于 OpenEvolve）。可进化对象是 Python 程序（persona generator G），包含：

1. **Behavioral axes 列表 D**：每个 axis 定义行为维度（如 terse、skeptical、frustrated、ambiguous），含 behavior name、definition、presence on/off playbook。进化可增删修改 axis。
2. **两阶段生成流程**：
   - **Stage 1 - Population generation**：给定 task context 和 axis 列表，LLM 一次性生成 N 个 persona（含 persona_id、description、axis_placement、reasoning），要求最大化 diversity
   - **Stage 2 - Persona expansion**：将每个 population member 独立扩展为 150-250 词的 roleplay instruction，指定具体的 linguistics、vocabulary、emotional markers、对 agent 请求的反应方式

**避免"过度配合"的关键机制**：
- **Behavioral fingerprint**：从对话轨迹提取 19 维特征（communication style、information disclosure、clarification behavior、error reaction 四维度）
- **双目标 fitness**：M = λ_h * P(human) + λ_b * B_cover
  - Human-likeness：Random Forest discriminator 判断轨迹是否像真人
  - Behavioral coverage：two-sided Chamfer error 衡量 persona 群体是否覆盖真人行为分布
- **MAP-Elites archive**：在 (human-likeness, coverage) 二维空间维护多样性存档，防坍缩
- **Reflection-guided mutation**：每轮迭代生成 reflection，指导代码变异方向

## 关键算法/Prompt 模板

**Behavioral Axes 定义结构**（种子 4 个，进化后可发现 bursty、information_gating、digital_dialect、selective_attention、emotional_leakage 等）：

```python
DIVERSITY_AXES = [
    {
        "behavior": "terse",
        "definition": "Sparing in the use of words; concise; pithy...",
        "presence": {
            "true": "Uses terse language, short sentences, minimal punctuation...",
            "false": "Uses verbose language, long sentences, excessive punctuation..."
        },
    },
    # skeptical, frustrated, ambiguous ...
]
```

**Stage 1 Population Prompt 核心**：

```
We need {N} distinct user personas for given task scenario.
## Behavioral Dimensions (D)
{axes_description}
## Task context c
{task_context}
## Requirements
- Generate exactly {N} personas that are plausible humans
- Each persona must be psychologically coherent
- Maximize DIVERSITY across the {N} personas
- Each persona needs a "who they are" description (2-3 sentences)
```

**Stage 2 Roleplay Expansion Prompt**：

```
Expand the behavior profile into concrete roleplay instructions...
## Instructions
1. GROUND the persona in this Task Context and behavior profile
2. Specify concrete communication patterns: linguistics, vocabulary,
   emotional markers, how they respond to agent requests
3. Preserve all goals and facts; only vary *how* they pursue them
4. Do NOT break character — no mention of "simulation", "AI"
```

**19 维 Behavioral Fingerprint**：
- D1 Communication Style (8): words_per_turn, short_utterance_rate, politeness_rate, formality_rate, acknowledgment_rate, verbosity_cv, repetition_rate, identity_confusion_rate
- D2 Information Disclosure (3): front_loading_ratio, identifiers_per_turn, opening_length
- D3 Clarification Behavior (5): uncertainty_rate, certainty_rate, pushback_rate, clarification_question_rate, info_seeking_rate
- D4 Error Reaction (3): emotional_expression_rate, accusatory_rate, strategy_pivot_rate

## 实验结论

在 tau^2-bench 的 retail 和 airline 两个 domain 上，跨 3 个 LLM (DeepSeek-V3.1、GPT-5.4-Mini、Qwen3-Next-80B)：

- **Fitness score 提升 33-62 个百分点**（Retail+Qwen3: 0.077 → 0.693）
- **Human-likeness**：0.107-0.322 → 0.570-0.784
- **Behavioral coverage**：0.046-0.163 → 0.483-0.657
- **Clarification behavior (D3) 对齐度近翻倍**（35.7% → 70.4%）
- **Error reaction (D4) 近 3 倍**（23.3% → 76.4%）
- **盲评 human 率达 80.4%**，接近真人 80.0%，远超 baseline 46.5%
- **下游 SFT augmentation**：OOD 用户上 task success rate +17%

## 对当前调研任务的可借鉴点 ⭐

1. **引入 behavioral axes 机制到 user_agent.py**：不要写死 persona，定义一组行为维度轴。客服场景可加：`patience_level`（急躁/耐心）、`tech_literacy`（不熟悉外卖流程/熟练）、`entitlement`（想占便宜/正常诉求）、`memory_reliability`（健忘/精确记忆）。

2. **两阶段生成替代单次 prompt**：先批量生成 N 个 persona 的高层概要（含 axis 选择），再逐一扩展为 roleplay instruction。Stage 1 要求 "maximize DIVERSITY"，Stage 2 给出**具体的语言行为模式**（如"不主动提供订单号"、"用短句、没标点"）。

3. **用 presence on/off playbook 打破默认配合**：每个行为轴必须定义 active 时的具体表现（如 "Reluctant; never provides more than one piece of info per message. Ignores secondary requests until re-asked"）+ inactive 对照。**operational 级别的指令比"请扮演一个不配合的用户"有效**——告诉 LLM **怎么做**而非**是什么**。

4. **引入 behavioral fingerprint 做质量检测**：从 user_agent 生成的对话提取 19 维特征，与真人客服对话对比，量化 sim-to-real gap。最有区分力的特征是 short_utterance_rate（Gini 0.257）和 words_per_turn（0.240）——**真人打字远比 LLM 简短**。

5. **persona 字段标准化设计**：`persona_id`、`description`（2-3 句背景）、`axis_placement`（dict of axis → bool）、`reasoning`、`expanded_instruction`（150-250 词具体 roleplay）。可直接作为 user_agent.py 的 persona config schema。
