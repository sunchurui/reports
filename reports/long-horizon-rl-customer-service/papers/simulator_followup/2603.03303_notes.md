# 2603.03303 — HumanLM: Simulating Users with State Alignment Beats Response Imitation

**作者**：Shirley Wu, Evelyn Choi, Arpandeep Khatua et al. (Stanford/NYU/Accenture)
**来源**：arXiv 2603.03303, 2026-03
**调研主题**：客服对话仿真系统的 user_agent 设计

## 一句话总结

训练 user simulator 时，与其让模型模仿用户的表层回复文本（response imitation），不如先显式生成用户的心理潜在状态（belief/goal/stance/emotion/communication），再基于这些 latent states 合成回复，对齐效果提升 16.3%。

## 核心方法

**核心论点**：Response imitation（vanilla SFT 直接拟合用户回复）只学到表面语言模式（emoji 频率、用词风格），无法捕捉驱动用户回复的深层心理状态。State alignment 的思路：先让模型预测用户在 6 个 state dimensions 上的 latent states，再从 states 合成回复。

**6 个 State Dimensions（心理学驱动）**：
- **Cognitive aspect**：belief（对世界的底层假设）、goal（本次交互想达成什么）
- **Normative aspect**：value（认为什么重要）、stance（对具体话题的立场）
- **Affective aspect**：emotion（带强度和对象，如 "moderate heartbreak for victims"）
- **Linguistic aspect**：communication（表达方式，仅训练时使用）

**训练流程**（两阶段）：
1. **Latent state generation**：给定 user profile + context + state dimension，模型生成对应自然语言 latent state。用 LLM judge batch scoring（0-1）衡量一致性。
2. **Response synthesis**：模型生成 reasoning trace（含所有 state dimensions），再合成最终回复。用 LLM judge 打 response alignment score。

**与 vanilla SFT 的关键区别**：
- SFT 直接最小化 ground truth 的 next-token loss → 学到表层模式
- HumanLM 用 GRPO 做 RL，reward 信号来自 state alignment scores 而非 token-level likelihood
- 训练时随机采样 state dimension，多个 rollouts 做 batch comparison，避免 LLM judge 的绝对评分偏差

## 关键公式/算法

**形式化目标（Eq.1）**：给定 input context x 的 latent state space S(x) = {s1, s2, ...}，对任意生成回复 y_hat，目标是最小化与 ground truth y 的 latent state 差异：

```
min_{y_hat} sum_{j=1}^{|S(x)|} |I(s_j ∈ M(y_hat)) - I(s_j ∈ M(y))|
```

其中 I 是 indicator function，M(y) 是从回复 y 提取 latent states 的映射。本质上把 response 看作 latent states 的 bag，惩罚缺失或多余。

**RL 训练**：GRPO (Shao et al., 2024)，group size = 4，batch size = 32。Reward 分两路：
- 生成 latent state 时：对应 state dimension 的 alignment score 作为 reward
- 生成 response 时：response alignment score 作为 reward

**评分公式（Appendix E.2）**：LLM judge 对生成提取 K 个 key points，逐一打分 m_i ∈ [0,1]，coverage C = (m_1+...+m_K)/K，加 penalty P 惩罚冗余/矛盾，score = max(0, min(1, C - P))。

**Prompt 模板**：模型以 `<think>...</think>` 格式生成 reasoning trace（包含对所有 state dimensions 的推理），再输出 `<response>...</response>`。训练时每个 state dimension 有独立 XML tag prompt（`<belief>`, `<goal>`, `<stance>`, `<emotion>`, `<communication>`），附详细描述。

## 实验结论

**Benchmark**：HUMANUAL —— 6 个数据集，26k 用户，216k 回复，覆盖新闻评论、书评、观点讨论、政治博客、多轮对话、邮件。

**主要结果**：
- HumanLM 平均 response alignment score = 13.2，比最强 baseline (GRPO-think 10.4) 相对提升 **16.3%**
- 比 SFT (6.5) 提升 **103%**，比 base Qwen3-8b (9.5) 提升 39%
- 在 6 个数据集上全部最佳，Humanual-Book 上提升 36%

**SFT 的致命缺陷**：SFT 在大规模训练后反而是所有方法中最差的（avg 6.5），因为过拟合表面语言模式而遗失高层状态信息。

**真人评测**（111 人）：HumanLM overall similarity 6.5/10，win rate 41.4%；76.6% 参与者认为"quite natural"或"indistinguishable from humans"。

**训练动态**：HumanLM 通过显式 state generation 获得更广的探索空间，state 和 response scores 的覆盖范围分别高出 23% 和 104%。

## 对当前调研任务的可借鉴点 ⭐

1. **客服场景的 user state dimensions 定义** — 论文的 6 维框架可直接迁移：
   - **goal**：用户本次诉求（退款/催单/投诉/咨询）
   - **emotion**：带强度的情绪（如 "moderate frustration about delivery delay"）
   - **stance**：对客服已提供方案的态度（接受/犹豫/拒绝）
   - **patience_remaining**：建议显式建模为 0-1 连续值
   - **clarity**：诉求表述清晰度（模糊抱怨 vs 明确要求）
   - **belief**：用户对平台规则的预期假设

2. **不重训模型、用 prompt 工程模拟 state alignment** — 在 user_agent.py 的 system prompt 中加 `<think>` 段，要求模型在回复前先显式推理 user state。每个 state dimension 用 XML tag 模板（`<goal>用户想要...</goal>`）。**关键：要求模型先输出 states 再输出 response**，而非反过来。

3. **State 对齐的验证方法** — LLM judge batch scoring：收集真实客服对话作 ground truth，对 user_agent 生成的回复逐维度打 alignment score。采用 batch comparison 避免绝对评分偏差。重点关注 goal 和 stance 维度（Pearson r 分别 0.66 和 0.59）。

4. **User profile 构建** — 论文用 claude-4.5-haiku 从用户最早 20 条回复中提取 structured profile，可用于从历史客服记录为 user_agent 构建 persona template。

5. **避免 SFT 陷阱** — 论文最重要的负面结论：**SFT 在大量数据上训练后效果反而最差**（比 zero-shot base model 还差），因过拟合表面模式（如 emoji 使用频率）。意味着如果要 fine-tune user_agent，必须用 state-level reward 而非 response-level imitation。**不做 fine-tuning 的情况下，通过 prompt 要求显式 state reasoning 是更安全的路径**。
