一、行业案例库

覆盖 5 个有代表性的客服系统，按场景紧密度评分。

京东集团京东 JIMI / 言犀大模型
★★★★★ 同集团可直接借鉴
场景：京东商城售前售后（电商主站）
拟人化关键做法：
  1. 语气词（「哦~」「~」）+ 情绪反馈（「感谢您！」）
  2. IP 形象 + 昵称定制
  3. 用户画像驱动个性化（性格 / 爱好 / 习惯）
  4. 「场景 → 需求判断 → 答案」对话进化
  5. 言犀大模型把情感分析列为六大核心能力，识别愤怒/焦虑/满意并调整应答
判断：外卖场景的「催单/骑手」诉求强度远高于电商售后，话术需要重新设计。
证据：pconline · 人人都是产品经理 · IDCTalk · 53AI · 飞书云文档
竞品美团智能客服
★★★★★ 直接对标，整套架构可借鉴
场景：外卖售中 / 售后（催单是公开案例）
拟人化关键做法：
  1. 六大能力：问题推荐 / 理解 / 对话管理 / 答案供给 / 话术推荐 / 会话摘要
  2. 催单场景明确：实时取订单+骑手位置+ETA，先安抚后给方案
  3. 对话管理走 Task-flow 人工配置 + 语义匹配做分支决策（算法和业务解耦）
  4. 用 itemCF 建事件图谱做关联问题推荐（Task 间调度）
  5. 大模型时代引入语义+情绪识别，识别「生气/焦虑」触发安抚或自动转人工
  6. 答案由业务人员预配置话术，算法负责路由
判断：外卖业务一致，架构思路（Task-flow + 情绪路由）可整套借鉴。
证据：知乎-核心技术 · 知乎-技术实践 · 腾讯云 · CSDN-美团 Tech
同业阿里小蜜 / 店小蜜
★★★★ 框架可借鉴，话术差异大
场景：淘宝天猫电商售前售后（消费者侧 + 商家侧）
拟人化关键做法：
  1. 「智能 + 人工」拟人交互体验为产品愿景
  2. 双 11 智能解决率 93.1%，单日 200 万轮（2017）
  3. 两层架构：意图识别层（上下文 + 领域推理）+ 问答匹配层
  4. 多轮：意图触发后按预编排流程收集参数，调业务接口
  5. 个性化：个性化称呼、「猜你想问」（基于促销+用户偏好预测下一问）、服务管家（结果主动推送）
  6. 情绪能力三方向：客户情绪安抚 / 客服质量检测 / 情绪生成式语聊
  7. 数字人多模态交互探索
判断：「情绪三方向」框架可整体迁移；「猜你想问」对外卖催单是关键启发。
证据：知乎-情绪回复 · CSDN-技术整理 · IT 之家 · 商业新知-数字人
海外Intercom Fin AI
★★★★ Tone 预设值得产品化
场景：SaaS / 多行业客服（B2B 为主）
拟人化关键做法：
  1. 定位为「第一个提供人类品质服务的 AI Agent」，刻意区分 chatbot
  2. 可配置 Tone of Voice：Professional / Friendly / Playful 多种预设
  3. Persona：自定义名字+头像+行为边界（guidelines），透明告知「我是 AI」
  4. 自适应：识别情绪后动态调措辞（不满时更体贴）
  5. 共情先于答案：识别到沮丧/困惑时先说「I understand this is frustrating」
  6. 平滑转人工：超出能力或情绪需要时转人工，附完整上下文
判断：Tone 预设和 persona 边界两件事是 SaaS 标配，外卖可直接做「语气包」产品化。
证据：Intercom 官网 · Fin AI Agent
海外Sierra AI（Bret Taylor 创立）
★★★★ 护栏 + 按结果付费值得借鉴
场景：跨行业（SiriusXM / ADT 家防等大客户）
拟人化关键做法：
  1. Bret Taylor 公开反 chatbot 概念，主推「empathetic AI agent」
  2. 品牌个性化人设：每个企业的 AI agent 有独特品牌人设（语气 + 性格）
  3. Guardrails 护栏机制：敏感场景（如丧亲请求）必须有规则防止幻觉编造政策
  4. 按结果付费商业模式（成功解决才收费）反向驱动质量
  5. 强调「用户实际很喜欢这些 agent」——共情设计直接体现在 NPS 和留存
判断：「敏感场景护栏」对外卖很关键（食品安全/中毒/吵架等）；「按结果付费」启发 KPI 应从「接通率」切到「解决率+满意度」。
证据：TechCrunch Disrupt · 腾讯新闻-Bret Taylor 访谈 · Agent 按结果付费

二、四维度做法对比

2.1 语言 / 语气

京东 JIMI：语气词收尾（「哦~」），引导词去机器化（「我来助你一臂之力~」），正向情绪反馈（评价成功后补「感谢您！」）。启示：拟人化首先是文案层微改造，成本极低。

美团客服：「话术由业务人员预配置 + 算法负责路由」的解耦模式——话术拟人化优化在业务侧。启示：要把话术运营 KPI 拎出来（如「机器人口吻满意度」）。

阿里小蜜：个性化称呼 + 生成式语聊（不是纯模板）。启示：模板话术 + 生成式润色 是目前最务实的方案——业务方写骨架保证准确，LLM 加修饰保证自然。

Intercom / Sierra：Tone 做成可配置预设；Sierra 强调每个品牌独特人设 + guardrails。启示：京东外卖应该有场景化语气包（催单耐心、投诉共情、骑手简洁），不能一套打天下；同时敏感场景必须护栏（食物中毒、过敏、火灾等极端反馈，绝不让 LLM 自由发挥）。

2.2 情感识别与共情回应

京东言犀：情感分析是六大核心能力之一，识别愤怒/焦虑/满意。启示：关键不在「识别」，在「识别完之后做了什么」——是否真换了话术分支、是否提速转人工、是否做了对账。

美团：实时监控情绪，触发自动安抚 or 自动转人工。催单标准动作：「先安抚 → 给订单状态 → 给可执行方案」。启示：「情感识别 → 触发动作」闭环必须明确，至少 3 个触发动作：A. 切共情话术；B. 提速 ETA / 通知骑手；C. 直接转人工。

阿里小蜜「情绪三件套」：客户情绪安抚 + 客服质量检测 + 情绪生成式语聊。启示：外卖共情体系建议按这三层拆。「质检」这层国内大多客服没认真做，是差异化点。

Intercom / Sierra：识别情绪后动态调措辞 + 共情先于答案 + 平滑转人工。启示：海外的共识比国内更激进——共情不是「加一句安抚」，而是整个对话从开场就根据情绪走完全不同的路径。京东外卖催单可以做：情绪在第一轮就分流，「正常催单」和「暴怒催单」走两套完全不同的话术树+动作树。

2.3 对话节奏与多轮交互

JIMI 三代演进：关键词→答案 (2012) → 语句→答案 (2012.12) → 场景→需求判断→答案 (2014.5)。启示：节奏拟人化的前置条件是「场景级需求判断」，而不是单轮 QA。

美团 Task-flow 架构：四层模块清晰，多轮做 Task 内调度 + Task 间调度（itemCF 事件图谱推荐关联问题）。启示：节奏不是「等用户来问」，而是「预测下一个问题主动给」。外卖催单可做：用户问完单状态后主动追问「是不是着急？需要我帮您催骑手吗？」。

阿里小蜜「猜你想问」+ 服务管家：预测下一问当 chip 摆出 + 售后流程结束后主动推送。启示：节奏拟人化最高境界是「我比你早一步」。外卖可做：(1) 用户输入还没发出但识别到关键词时预加载；(2) 骑手送达后主动追问「味道还好吗？」；(3) 退款审核通过主动推送。

Intercom / Sierra：两家都强调「AI agent」而非 chatbot，本质是希望主动推进。Sierra「按结果付费」反向要求 agent 主动闭环。启示：外卖客服 KPI 应分出「机器人首轮解决率 / 主动追问转化率 / 用户中断后挽回率」三个细分。

2.4 评测与度量

行业共性问题：公开资料里 JIMI 只报「大促单日 130 万+ 接待量」这类规模指标，没看到拟人性的量化评测——这是差异化点。

美团暗示的评测维度（从架构反推）：「情绪识别准确率」+「转人工时机」+「话术采纳率」。

阿里小蜜可量化指标：双 11 智能解决率 93.1%、服务占比 95%、单日 200 万轮+。「客服质量检测」隐含一个指标：客服服务后用户情绪是否变好（pre→post 情绪极性差）。

Sierra「按结果付费」 = 把「解决率」作为终极指标。启示：外卖客服拟人性评测应包含 4 个指标：(1) 解决率（不靠转人工的真实解决）；(2) 情绪转化率（负→中/正）；(3)「你是机器人吗」触发率（越低越拟人）；(4) CSAT / NPS。

三、京东外卖现状与差距诊断

3.1 现状速写（公开资料拼图）

集团基础：京东集团 2017 年起电商主站全线接入 JIMI，11.11 机器人接待占比近 90%、独立解决率 60%+。2023 年起对外推言犀大模型，情感分析为六大核心能力之一。

外卖业务实际表现（公开投诉里反复出现的痛点）

  • 「退押金难，全程机器人通话」（黑猫 2025-09）
  • 「骑手客服是机器人，一直联系不上」（黑猫 2026-01）
  • 「客服搪塞问题，和机器人有什么区别」（黑猫）
  • 拒不出餐场景「京东客服仅回复了一条模板化的引导消息，让用户自行通过 APP 联系在线客服」（黑猫 2025）

行业共性翻车（外卖业务集体患病）

用户气急说「AI 退退退」，AI 直接执行退款；用户怎么挽回 AI 都只会「道歉+补 5 元红包」——证明语义理解和情感反馈都没做到位（腾讯新闻 2025-08）。

结论：京东外卖客服机器人体验明显滞后于京东电商主站，集团有能力但没下沉到外卖业务；投诉里直接点名「机器人话术」是核心抱怨。

3.2 差距点（按四维度归类）

差距 1：语气产品化缺失 — 「机器人话术」是用户直接抱怨的词

对标：Intercom Fin 多语气预设；阿里小蜜个性化称呼 + 生成式润色；JIMI 主站靠语气词和情绪反馈做文案级拟人。

现状：投诉文本里「机器人话术」是直接被用户当成贬义词在用——证明语气没分场景、没拟人化、读起来就是模板。证据：案例 1 · 案例 2。

本质：话术运营没有按「催单 / 退款 / 骑手联络 / 投诉 / 异常」等场景做分语气包。

差距 2：情绪 → 动作闭环缺失 — 识别了情绪也没差异化处置

对标：美团情绪识别触发安抚 or 自动转人工；阿里小蜜情绪三件套；Sierra 在敏感场景（丧亲）有 guardrails 防止幻觉。

现状：「机器人只会道歉+补 5 元红包」——即使识别到负面情绪，触发动作的多样性严重不足。腾讯新闻案例里 AI 重复「非常抱歉」+5 元补偿陷入死循环；同时出现「AI 退退退」被误判退款。

本质：情感识别和业务动作没解耦——应走「先安抚 → 给 ETA → 主动催骑手 → 转专员」四步动作链，而非只切一句道歉模板。

差距 3：节奏被动 — 只会等用户来问，不会主动推进

对标：阿里小蜜「猜你想问」预测下一问、「服务管家」主动推送；Sierra agent 主动闭环；美团 Task 间调度做关联推荐。

现状：投诉里反复「打不通」「联系不上」「让我自行通过 APP 联系」——机器人在用户首问后既不主动追问、不给后续状态、也不主动转人工。

本质：缺少主动推进机制——(a) 首问后没主动追问；(b) 订单异常没主动推送；(c) 转人工决策权交给用户而非 bot。

差距 4：评测指标错位 — 只看解决率/接通率，不看拟人性/情绪转化

对标：Sierra 按结果付费倒逼解决率+满意度；阿里小蜜「客服质量检测」反推出情绪 pre→post 转化率指标。

现状：机器人「答非所问」但话术上仍闭环（KPI 上算「已应答」），用户感受和指标严重背离。

本质：缺少拟人性度量——「你是机器人吗」触发率、用户骂人比例、负面情绪转化率、首轮真实解决率（剔除模板应付）都没作为日常监控。

四、仿真环境下的拟人性评测

注：项目当前数据来自仿真环境（用户输入由脚本生成），「负面情绪转化率」这类依赖真实用户反应的指标暂时失效。仿真阶段的评测目标是"bot 输出是否符合拟人化设计规范"，等上线后再叠加情绪转化等真实反应指标。

4.1 LLM-as-Judge 五维打分（主力方案）

用评判 LLM 对每条 bot 回复打分，五个维度：

维度判断要点输出
自然度这句话像真人客服会说的吗？1-5 分
共情命中用户上一句若含负面情绪，bot 有没有先共情再给方案是 / 否
机器人腔检测是否包含「非常抱歉给您带来不便」等八股套话是 / 否
主动推进该主动时（追问/推送/转人工）有没有主动是 / 否
事实准确订单状态、ETA、退款规则是否对是 / 否

适合大规模、可重复跑，几千通会话半天能出结果。

4.2 规则检测（兜底 + 低成本）

  • 重复道歉：一通会话「非常抱歉」> 2 次 → 标红（「AI 退退退」案例的核心特征）
  • 同句重复：bot 连续两轮说同一句话 → 标红
  • 共情命中：用户出现「着急」「快点」「恼火」后，bot 下一句是否包含共情词
  • 敏感词护栏：用户提「中毒」「过敏」「烫伤」等，bot 必须立即转人工（100%）
  • 句长与标点分布：句子过长、句末无标点 / 缺语气词偏离真人语料

4.3 真人对照指纹（差异化）

用历史真人客服会话作为基线，统计：句长分布、语气词频次、感叹号比例、表情/emoji 使用率。bot 输出统计与真人偏离越大，越不像人。外卖客服特别合适——历史真人会话存量大、标注成本几乎为零。

4.4 LLM 模拟用户（进阶，待真人数据接入后再考虑）

把仿真用户从脚本改成 LLM 扮演，根据 bot 上一句的语气动态调整自己下一句的情绪——这样「情绪转化率」就重新有意义了。但成本高、引入新变量（评测耦合于另一个 LLM），不建议作为第一步。

4.5 上线后再接的真实反应指标

  • 「你是机器人吗」触发率（用户主动质疑身份）
  • 负面情绪 pre→post 转化率
  • 用户骂人/不文明用语比例
  • 会话中断率（用户中途关闭）
  • 平均轮次到解决

4.6 推荐落地组合

阶段方法成本价值
仿真阶段 P0LLM-as-Judge 五维1 周搭好强
仿真阶段 P0规则检测（重复道歉 / 同句重复 / 共情命中 / 敏感词护栏）2-3 天强
仿真阶段 P1真人对照指纹1-2 周中、差异化

五、改造优先级表

优先级判据：性价比 = 预期效果 ÷ 实施成本。P0 = 1 个月内可上线，P1 = 1 季度内，P2 = 半年以上。「需内部核实」表示该项基于公开投诉推断现状，落地前要看京东外卖内部话术/对话日志校验。
优先级 改造点 对应差距 预期效果 实施成本 风险
P0-1 场景化语气包：催单 / 退款 / 骑手联络 / 投诉 / 异常 5 个场景各做一套话术骨架，由话术运营维护；LLM 仅做风格润色不改事实 差距 1 语气产品化 投诉「机器人话术」高频词预计下降；最易感知 低（话术工程 + 1 周配置） 低：仅文案改造，可灰度回滚
P0-2 敏感场景 guardrails：食物中毒 / 过敏 / 烫伤 / 异物 / 吵架升级 等强制走「快速共情 + 立即转人工 + 订单冻结」硬规则，禁止 LLM 自由生成或自动操作 差距 2 + 差距 4 杜绝「AI 退退退」类事故；规避公关风险 低（关键词 + 规则引擎，1-2 周） 极低，反而降低存量风险
P0-3 仿真阶段评测体系：LLM-as-Judge 五维 + 规则检测 + 真人对照指纹；上线后接真实反应指标 差距 4 评测错位 拟人性优化有量化抓手，作为后续所有改造的 A/B 评判 低（1-2 周） 低，纯观测
P1-1 情绪 → 动作 4 步链：① 切共情话术 → ② 调订单接口提速 ETA / 催骑手 → ③ 分级补偿（不是固定 5 元）→ ④ 主动转专员 差距 2 闭环 解决「道歉 + 5 元」死循环；用户感知「系统真的在帮我」 中（情绪模型 + 动作编排 + 补偿规则审批，1 季度） 中：补偿规则需财务/风控审批，需防刷
P1-2 主动节奏三件套：(a) 首问后主动追问「是不是着急？」；(b) 订单异常主动推送；(c) bot 自主决定转人工 差距 3 节奏被动 「打不通 / 联系不上」类投诉本质解决 中（推送通道 + 主动逻辑 + 人工排队联动，1 季度） 中：主动推送频次需把控防骚扰
P1-3 模板 + LLM 润色 双层话术：业务方写骨架保证准确，LLM 做语气改写保证自然；配「机器人口吻评分」事后质检 差距 1 深化 + 差距 4 质检 话术不僵且不出事实差错 中（LLM 微调 + 离线评测平台，6-8 周） 中：LLM 幻觉风险，需对照事实校验
P2-1 个性化人设产品化：基于用户标签（新老用户、VIP、投诉历史）切不同人设包 差距 1 + 差距 2 高价值用户感知差异化；提升留存 高（标签打通 + 多人设维护 + 权限管理） 中：标签错配带来歧视感、合规风险
P2-2 情绪质检反哺人工座席：同一套情绪模型给人工座席做事后质检，「AI 检 AI、AI 检人」统一闭环 差距 4 评测 + 差距 2 对内 拟人化从对外延伸到内部质量管理 高（数据打通 + 座席系统改造 + 工会沟通） 高：涉及员工绩效，需 HR/工会沟通

5.1 落地路线建议（按节奏）

第 1 个月

第 2-3 个月
